1. Introduction

Roim is a class library designed to help implementing software for processing nuclear medicine (hereinafter NM) data. This development was inspired by ImageJ image viewer and written in Java. The library is designed to be vendor neutral and shall support most of modalities. However, for the time being it is tested only with NM/DR/DXR/CT/PET/MR files. It implements a set of controls, screen primitives (Overlays) and algorithms to extract, process and display biomedical information embodied in study files.
Namely:
  • regions of interest (hereinafter ROI) of rectangular, elliptic and free form shape - once ROI is created it can be moved, reshaped, deleted, pinned, etc and a set of characteristics can be measured. Among them area (in pixels and cm), integral density (i.e. arithmetic sum of pixel values), min/max/avg pixel. In the case of multiframe image - often seen in NM - this information will be extracted out of all the frames and then one will have a possibility to build and display curves like "density over time";
  • Ruler - it is meant for extracting geometrical information of the images;
  • Profile - a tool to display density profile over a rectangular area of an image;
Beyond aforementioned tools there is a number of instruments to adjust visualization parameters W/L (Window Width and Level) and apply presentation LUT (a dozen most popular ones is built into and there exists a possibility to load LUT from files in NIH format). Frame selection bar provides a mechanism of navigating through multiframe/multi-phase images in either terms of frames or time (milliseconds).
  • Using ROI calculator it is possible to apply basic maths to measurements. For instance: one of often necessary operation is subtraction of background counts of counts measured from a ROI. Then, the resulting series can be send to a chart control.
  • Chart control is built around JFreeChart library and is designated to visualize series as curves, extract numeric information and finally
  • there is an instrument to fit experimental data with linear or exponential polynomials using least squares method. It is not seldom acquisition ends earlier than a particular event takes place, for instance a half excretion time in the case of renal obstruction. Using this tool it becomes possible to extrapolate experimental data in many cases achieving acceptable approximation of missing parameter.
Visualization provides different rendering methods namely nearest neighbourhood, bilinear and bicubic it is especially important since NM images usually have quite low resolution, thus, zooming them in would look quite unpleasant on the screen. Reconstructed SPECT (3D) studies can be displayed either as a stack of slices (usually transaxial) or using MIP method. The library relies on DCM4CHE to handle DICOM data and aparapi to employ GPU based data processing. Finally the library includes several applications to illustrate basic concepts and principles behind the design.

2. LUT Control

The picture below shows LUT control in the action with open menu. A number of options are available to made reading as much comfortable as possible. Beyond selection of LUT it is possible to use logarithmic function to translate image pixels to screen luminosity (actually sigmoid is used), inverted scales and yet there is a form displaying image's histogram accompanied with live W/L translation curve. Window width and level can be adjusted using control using knobs on the bar control, displayed values are either in arbitrary units or percents.

3. Frame Selector

The Frame selector is meant for navigating through the image. Depending on image type it measures either time or space. Time scale can show milliseconds from either beginning of the whole study or a particular phase (for multiphase studies only). Of course, there is a mode in which it operates purely with frames.

4. Application

This application is meant for dealing with data of dynamic acquisitions such as renal clearance or gastric emptying studies. The screen is split into two parts left one is tab control displaying image as a stack of frames (shown), composite image and grid of frames in subsequent tabs. Right part is occupied the chart control displaying activity curves of two ROIs and extrapolation of one of them. Beyound there are two more applications embedded: namely VIEWER and TOMO. The former is pretty simple image viewing aid while the latter is capable of showing 3D series as MIP. It even can open multiimage series placed in the same location.
SIC: application requires JDK 8 to run

5. Images
